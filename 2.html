  • 2. 코딩의 첫머리 규칙


    • 저도 잘 모르겠지만 코딩의 첫 시작은 html이라는 언어로 시작을 할 것이며, Visual studio code라는 친구와 함께 공부를 시작할것입니다.
    해당 친구는 코딩 html을 짜주는 프로그래밍입니다.

    < > 이런 기호 안에 영어 약자나 단어를 사용하여 문자를 특수한 효과를 나타내는 것이 tag(태그)입니다.
    그렇다면 우리는 코딩의 첫머리 규칙에 대해서 알아볼텐데요!


    < !DOCTYPE html >

    이유는 저도 모릅니다!!!! 부끄
    아마 이 doc (document = 문서)의 type(형태)은 html으로 이루어졌다 라는걸 먼저 알려주는게 아닐까 싶습니다.
    책 작성할때 저자가 누군지 밝히는거와 같이 말이죠?!
    word파일이 doc와 같은 느낌으로 작성하는게 아닌가 싶습니다.
    자세한 내용은 아마 html을 처음으로 작성한 사람에게 물어봐주세요 ㅎㅎ
    이것을 먼저 첫 시작으로 넣고, 그다음에는

    < html >

    이라는 친구를 넣어줍니다.
    역시나 다시한번 html의 형식으로 글을 작성한다는것을 밝히는 것 같습니다.

    < head >

    다음으로는 head 입니다.
    대략 위의 내용을 묶어서 작성하기위한 태그라합니다.

    < title >

    이 친구는 사이트의 이름을 나타내주는 친구입니다.

    이렇게 웹사이트 엔진에서 이름을 나오게 도와줍니다.

    < meta charset="UTF-8" >

    우와! 매우 이름이 어렵군요.
    저도 자세하게 뭔지는 모르지만...ㅠㅠ
    대충 듣기로는 html파일을 어떤 형식으로 이용해서 열거라고 들었습니다.
    이것도 전문가에게 물어보는것을 추천드립니다..지송!!
